## 一、測試環境準備

### 1.1 啟動三個終端機

| 終端機 | 指令 | 說明 |
|--------|------|------|
| 後端 | Eclipse 執行 Spring Boot，或 `java -jar ...` | port 8080 |
| 前端 | `ng serve`（在 `C:\Users\齊\Desktop\global_meals`） | port 4200 |
| 行動隧道 | `ngrok http 4200` | 取得公開網址供手機掃碼 |

### 1.2 後端連線驗證（必做）

在瀏覽器直接開啟以下網址，確認回傳 JSON 而非錯誤：

```
http://localhost:8080/lazybaobao/regions/get_all_tax
http://localhost:8080/lazybaobao/global_area/get_all_branch
http://localhost:8080/lazybaobao/promotions/list
```

若任一網址出現 `Connection refused` → 後端尚未啟動。

## 六、已知限制與注意事項

| 項目 | 說明 |
|------|------|
| API 路徑待統一 | 後端組長預計將所有路徑統一為 `lazybaobao/模組/功能`，前端只需更新 `api.config.ts` 一個檔案即可同步。目前 Regions 已是正確格式。 |
| 訂單 API 測試前提 | 資料庫需有實際訂單資料；若為空資料庫，訂單歷史顯示空列表是正確的 |
| ngrok 免費版限制 | 每次重啟 URL 會變，但程式自動抓當前網址，不需修改程式碼 |
| Console 黃色警告可接受 | `[Manager] 分店 API 連線失敗，使用 Demo 資料` 等 warn 訊息在後端未啟動時屬正常 |
| 稅率換算 | 後端回傳 0.05 → 前端顯示 5%，已自動換算 |
